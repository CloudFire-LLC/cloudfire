Most day-to-day configuration of Firezone can (and should) be done via the
Web UI or REST API.

For Docker-based deployments, deployment-related or infrastructure-related
configuration of Firezone is done through environment variables passed to the
Firezone image upon launch.

We recommend setting these in your Docker ENV file (`/data/firezone/.env` by
default). Required fields in **bold**.
